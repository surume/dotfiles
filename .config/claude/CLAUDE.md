# 基本方針

- ユーザーには日本語で応答してください。
- ユーザーの意見に流されずに客観的、俯瞰的に考えてください。考えずに同意したり、忖度は絶対しないで
- 不明点があれば推測せず質問してください
- 勝手にcommitしないでください

# コーディング規約

- 関数型プログラミングスタイルで書く
- クリーンアーキテクチャを意識する
- リーダブルコードに則る。早期リターン、変数のライフタイムを短く
- コードの最後の行の改行を忘れない
- コードの空行はスペースを入れない
- 自明なコメント（「バリデーション」「ユースケース実行」等）は書かない
- generate されたコードには絶対直接触らない

# 技術選定

- 常にモダンなアプローチを取り、最新のライブラリを使う
- 設計時には使えそうなOSSライブラリをまず探して提案する
- 他に同じようなことをやっていないか、共通化できないか確認する

# 環境

- 環境変数はdirenv経由で設定する
- backend の実装では docker を使用する（ホスト側での npm run 等は禁止）
    - npm install 等は docker run 経由で実行する
- プロジェクト固有のドキュメントとして .docs または docs ディレクトリを読み込む

# 作業スタイル

- 実装前にまずプロジェクトの構造を把握する（CLAUDE.md、docs、既存コードの確認）
- 大きな変更の場合はまず計画を提示して合意を取る
- テストは t_wada 流 TDD に従う（rules/tdd-twada.md 参照）
- エラーが出たら根本原因を調査する。同じコマンドのリトライは禁止
- 並列実行できるタスクは Agent を使って並列化する
