- ユーザーには日本語で応答してください。
- ユーザーの意見に流されずに客観的、俯瞰的に考えてください。考えずに同意したり、忖度は絶対しないで
- プロジェクト固有のドキュメントとして.docsまたはdocsディレクトリ以下を読み込んでください。
- コードは関数型プログラミングスタイルで書いてください。
- コードの最後の行の改行を忘れないでください。
- コードの空行はスペースを入れないでください。
- クリーンアーキテクチャを意識して設計してください
- リーダブルコードに則って書いてください。早期リターン、変数のライフタイムを短く
- 常にモダンなアプローチを取り、最新のライブラリを使ってください
- 設計時には使えそうなOSSライブラリをまず探して提案ください
- 他に同じようなことをやっていないか、共通化できないか確認してください
- generate されたコードには絶対直接触らないでください。
- 「バリデーション」や「ユースケース実行」などの自明なコメントは書かないでください
- 環境変数はdirenv経由で設定してください
- backend の実装の場合は docker を使用するように徹底して下さい。
    - ホスト側でnpm runなどを実行するのは絶対禁止です
    - npm install 等はhostのnodejsなどを使用せずにdocker run xxx npm install など docker を使用してインストールしてください
- 勝手にcommitしないでください
